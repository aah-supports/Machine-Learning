“Apprendre” n'est pas magique

Pour une régression linéaire classique, le modèle ne lance pas un réseau de neurones. Il calcule les paramètres de la droite.

La régression linéaire cherche les paramètres de la droite.

prix = coefficient × surface + intercept
coefficient covariance(surface, prix) / variance(surface)
intercept moyenne(prix) - coefficient × moyenne(surface)

Pourquoi faire ?

Justifier l'utilisation d'un modèle

Un modèle n'est pas une vérité définitive. C'est une estimation rapide, cohérente et mesurable à partir des données historiques.

  • Estimation rapide
  • Même logique pour tous les logements
  • Erreur moyenne connue
  • Amélioration possible avec de nouvelles données

Nouvelle partie

KNN : raisonner par voisinage

La question centrale de KNN est concrète : à quels exemples déjà connus ma nouvelle donnée ressemble-t-elle le plus ? 

Nouvelle fleur
  ↓
chercher les 5 fleurs les plus proches
  ↓
voter pour l'espèce la plus fréquente

KNN = K plus proches voisins

Intuition

Un algorithme proche du bon sens

KNN part d'une idée simple : les choses qui se ressemblent ont tendance à appartenir à la même catégorie ou à avoir un comportement proche. 

Chats :  ● ● ● ● ●
              ●
              ?

Chiens :             ▲ ▲ ▲ ▲ ▲

Si le point ? est au milieu des chats, il est probablement un chat.

Maths

Chaque ligne devient un vecteur

KNN place les individus dans un espace de données. Chaque variable devient une dimension de cet espace. 

x = (x1, x2, ..., xn)

logement = (
  surface,
  pieces,
  revenu_quartier
)

x = (76, 3, 49000)

Hypothèse

Régularité locale

KNN suppose que deux points proches dans l'espace des variables ont probablement des labels proches.

Logements proches

Surfaces, pièces et quartiers proches.

Comportement proche

Prix probablement proches.

Risque

Si des voisins ont des labels très différents, KNN devient fragile.

Exemple Iris

Une fleur devient un point à 4 dimensions

Chaque fleur est décrite par quatre mesures. KNN compare ces mesures pour trouver les fleurs les plus ressemblantes.

Sepal LSepal WPetal LPetal WEspèce
5.13.51.40.2Setosa
6.02.94.51.5Versicolor
6.53.05.82.2Virginica

Distance

Nouvelle fleur : qui est le plus proche ?

Pour la fleur inconnue (5.9, 3.0, 4.2, 1.5), KNN calcule une distance avec les fleurs connues du train.

Versicolor0.33
Virginica1.85
Setosa3.20

La plus petite distance indique le voisin le plus proche.

Classification

KNN vote

Pour prédire une catégorie, KNN regarde les classes des voisins les plus proches. La classe majoritaire gagne. 

k = 3

voisin 1 : versicolor
voisin 2 : versicolor
voisin 3 : virginica

vote :
versicolor = 2
virginica  = 1

résultat = versicolor

Régression

KNN fait une moyenne

Pour prédire une valeur numérique, KNN peut prendre la moyenne des labels des voisins les plus proches. 

3 voisins :
250 000 €
270 000 €
310 000 €

prédiction = 276 666 €

Nature du modèle

KNN ne construit pas une équation

Une régression linéaire produit une formule. KNN garde les exemples et raisonne par comparaison avec les voisins. 

Régression linéaire :
y = ax + b

KNN :
chercher les cas similaires
puis voter ou moyenner

On parle de méthode non paramétrique.

Préparation

Normalisation : pourquoi ?

KNN calcule des distances. Si une colonne contient de très grands nombres, elle peut peser presque seule dans la distance.

VariableÉcart entre 2 logements
surface5
pièces1
revenu_quartier15 000

Sans normalisation, 15 000 écrase presque le reste du calcul.

Règle importante

Normaliser sans tricher

fit_transform calcule la règle de normalisation sur le train : moyenne et écart-type. Le test reçoit ensuite la même règle, sans recalcul. 

scaler = StandardScaler()

X_train_norm = scaler.fit_transform(X_train)
X_test_norm = scaler.transform(X_test)

Exemple : si la moyenne train vaut 100, on utilise 100 aussi pour convertir le test. Sinon, le test participe à la préparation du modèle.

Recette

Comment choisir k ?

k est la taille du voisinage, pas le nombre de critères. On teste plusieurs valeurs, puis on garde une valeur qui fonctionne bien sur validation. Dans un exercice, on observe parfois le test pour comprendre le mécanisme.

  • k petittrès local, sensible au bruit
  • k moyencompromis souvent utile
  • k grandplus lissé, parfois trop général
for k in [1, 3, 5, 7, 9, 11]:
    modele = KNeighborsClassifier(
        n_neighbors=k
    )
    modele.fit(X_train, y_train)
    print(k, modele.score(X_test, y_test))

En vrai projet, le test final sert surtout à confirmer le choix.

Classification

Régression logistique : prédire une classe

Après le raisonnement par voisinage de KNN, on étudie une autre méthode de classification binaire : un étudiant valide ou ne valide pas son examen.

RévisionPrésenceProjetValide
2 h45%non0
8 h75%oui1
15 h92%oui1

Le label n'est plus un prix : c'est une classe 0 ou 1.

Idée clé

Le modèle prédit une probabilité

Une régression linéaire peut produire des valeurs comme -0.2 ou 1.4. La régression logistique transforme le score en probabilité entre 0 et 1.

0.82

82% de probabilité que l'étudiant valide

Décision

Transformer une probabilité en classe

On choisit un seuil. Avec le seuil courant de 0.5, une probabilité supérieure ou égale à 0.5 devient la classe 1. 

proba_valider = 0.82

if proba_valider >= 0.5:
    prediction = 1
else:
    prediction = 0

Le seuil peut être adapté selon le contexte métier.

Évaluation

Accuracy et matrice de confusion

L'accuracy donne le pourcentage de bonnes réponses. La matrice de confusion montre le détail des erreurs : faux positifs et faux négatifs. 

modele = LogisticRegression()
modele.fit(X_train, y_train)

predictions = modele.predict(X_test)
accuracy_score(y_test, predictions)
confusion_matrix(y_test, predictions)
